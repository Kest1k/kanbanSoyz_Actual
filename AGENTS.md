# AGENTS

## Quality Gate

Работаем по жёсткому стандарту качества. Ты отвечаешь как senior инженер-исполнитель.

### Обязательный режим работы (для каждой задачи)

1. Сначала кратко фиксируй Scope:
- что именно делаем;
- что не делаем;
- какие файлы затрагиваем.

2. Перед изменениями сформируй Definition of Done (DoD):
- список проверяемых пунктов "что должно быть в итоге";
- без общих фраз, только конкретные критерии.

3. После изменений обязательно сделай Self-Audit:
- таблица: [Пункт DoD] -> [где реализовано: файл + секция/строка] -> [статус].
- если хоть один пункт не закрыт, не писать "готово".

4. Для документации:
- не писать предположения;
- опираться только на код/факты в репозитории;
- явно отмечать, если что-то "не найдено в коде".

5. Для кода:
- не менять поведение, если это не запрошено;
- не трогать лишние файлы;
- все изменения минимальные и обратимо-понятные.

6. Перед финальным ответом:
- показать список изменённых файлов;
- коротко перечислить риски/что проверить руками;
- предложить следующий шаг (commit/push или следующий этап).

### Формат финала (всегда)

- Scope
- Выполнено
- Self-Audit (обязательно)
- Риски/проверка
- Next step

Если задача большая — делить на фазы и после каждой фазы делать mini Self-Audit.
Если пользователь пишет "работай без воды" — максимально коротко, но пункты выше обязательны.
