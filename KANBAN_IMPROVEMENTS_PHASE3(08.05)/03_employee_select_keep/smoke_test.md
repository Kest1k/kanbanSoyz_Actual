# Smoke-тест 03 — Сохранение фамилии в селекте «Все сотрудники»

Длительность: ~2 мин. Понадобится учётная запись руководителя (admin / headOfDept / headOfSector / leadEngineer).

## Подготовка

1. Залогиниться руководителем.
2. Открыть Канбан-доску.
3. Убедиться, что в верхней панели виден кластер «Мои задачи / Все задачи» и селект списка сотрудников.

## 1. Базовый сценарий

- [ ] В селекте «Все сотрудники» выбрать конкретного сотрудника (например, Иванов И.И.).
- [ ] Доска перерисовывается, на ней — только задачи Иванова.
- [ ] **Селект продолжает показывать «Иванов И.И.»** (не сбросился).
- [ ] Кнопка «Мои задачи» — синяя, не активная (бледнее).

## 2. Перезагрузка страницы

- [ ] Нажать F5 (или иначе перерисовать). Селект всё ещё показывает «Иванов И.И.».
- [ ] Закрыть Soyuz клиент, открыть снова → попасть на канбан-доску → селект показывает «Иванов И.И.».

## 3. Сброс через «Мои задачи»

- [ ] Нажать «Мои задачи». Селект сбросился на «Все сотрудники». Доска — мои задачи.

## 4. Повторный выбор

- [ ] Выбрать другого сотрудника (например, Петров П.П.).
- [ ] Селект показывает «Петров П.П.», доска — задачи Петрова.

## 5. Через поиск

- [ ] Ввести в поле поиска «Иван». В селекте — только Иванов И.И. и аналогичные.
- [ ] Нажать Enter (или кликнуть в селекте по нужному). Селект показывает выбранного сотрудника.
- [ ] Очистить поле поиска. Селект остаётся на выбранном сотруднике (не сбрасывается на «Все сотрудники»).

## 6. Регрессия — другие селекты

- [ ] Сменить отделение (для admin) — селект сектора и сотрудников перезаливается, фамилия в kb-sel-user сбрасывается на «Все сотрудники» (это норма — другой контекст).
- [ ] Выбрать сотрудника в новом отделении — фамилия закрепляется как и в шаге 1.
- [ ] Кнопка «Все задачи» работает: при клике селекты сбрасываются на «Все ...», виден весь скоуп.

## 7. Регрессия — обычный сотрудник

- [ ] Зайти под обычным сотрудником (regular). Панель иерархии вообще скрыта. Никаких регрессий.
