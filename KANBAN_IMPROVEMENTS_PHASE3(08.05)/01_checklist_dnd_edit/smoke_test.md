# Smoke-тест 01 — DnD и редактирование пунктов чек-листа

Длительность: ~5 мин. Понадобится одна задача с минимум 4 пунктами чек-листа.

## Подготовка

1. Открыть Канбан-доску.
2. Создать задачу «Тест Phase3 #01» (или открыть существующую).
3. Открыть карточку → вкладка «Чек-лист».
4. Добавить 4 пункта подряд: «Раз», «Два», «Три», «Четыре».
5. Отметить «Два» как выполненный.

## 0. Расположение поля ввода

- [ ] Сразу после клика на вкладку «Чек-лист» поле «Добавить пункт чек-листа...» **видно вверху**, под заголовком «Чек-лист 0/0». Курсор не нужно тянуть в самый низ.
- [ ] Под полем ввода — подсказка «Перетащите ≡ чтобы изменить порядок. Двойной клик по тексту — редактировать.».
- [ ] Под подсказкой — список пунктов (или «Нет подзадач» если пусто).
- [ ] Добавление пункта: ввёл текст, Enter — новый пункт уходит вниз, **поле ввода остаётся вверху**.
- [ ] При 20+ пунктах: список начинает скроллиться внутри вкладки, поле ввода и заголовок зафиксированы сверху и видны всегда.

## 1. Drag-and-drop

- [ ] Слева у каждого пункта виден значок-рукоятка `≡` серого цвета.
- [ ] При наведении мышкой на строку рукоятка становится синей.
- [ ] Захватить «Раз» за строку (или рукоятку) и тащить вниз. Виден полупрозрачный «дух» строки.
- [ ] При наведении на верхнюю половину «Четыре» — синяя линия **сверху** «Четыре»; на нижнюю — **снизу**.
- [ ] Отпустить над нижней половиной «Четыре». «Раз» переместился в конец, порядок: «Два», «Три», «Четыре», «Раз».
- [ ] Зайти на вкладку «История» — видна запись «Изменён порядок пунктов» с моим именем и датой.
- [ ] Закрыть карточку, открыть снова. Порядок сохранился.

## 2. Inline-редактирование

- [ ] Двойной клик по тексту «Два». Текст превратился в инпут с фокусом, текст «Два» выделен.
- [ ] Заменить на «Два с половиной». Нажать Enter. Текст обновился.
- [ ] На вкладке «История» — запись «Изменён пункт: Два → Два с половиной».
- [ ] Двойной клик по «Три». Стереть, ввести «Хм». Нажать Esc. Текст вернулся к «Три» (отмена).
- [ ] Двойной клик по «Раз». Стереть всё (пустая строка), Enter. Текст не сохранился (отмена через пустоту).
- [ ] Двойной клик по «Четыре». Кликнуть мышкой ВНЕ инпута (например, в шапку модалки). Текст сохранился (как Enter).
- [ ] Если изменили текст на тот же самый — в истории нет лишней записи.

## 3. Чекбокс/удаление не сломались

- [ ] Кликнуть по чекбоксу «Раз» — отмечается, бейдж `☑ N/M` обновляется.
- [ ] Снять отметку — бейдж обратно.
- [ ] Навести мышкой → виден «×» справа. Клик → подтверждение удаления → пункт исчезает.

## 4. Краевые случаи

- [ ] Перетащить пункт сам на себя — порядок не меняется, ошибки нет.
- [ ] Открыть редактирование, начать тащить ту же строку — DnD не запускается (пока редактируем).
- [ ] При длинном тексте (300+ символов) — текст разрывается по словам, рукоятка остаётся на месте.

## 5. Многопользовательский сценарий (опционально)

- [ ] Пользователь A открыл задачу с 4 пунктами.
- [ ] Пользователь B изменил текст пункта №2 через свою сессию.
- [ ] Пользователь A перетащил пункт №2. Сервер обработает, но текст у A пока старый. После перезагрузки модалки A увидит актуальный текст и новый порядок.
