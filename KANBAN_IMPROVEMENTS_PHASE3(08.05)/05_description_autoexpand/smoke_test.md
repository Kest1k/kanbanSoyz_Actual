# Smoke-тест 05 — Растягивание описания по содержимому

Длительность: ~3 мин.

## Подготовка

1. Открыть Канбан-доску.
2. Подготовить две задачи:
   - **Тест 05 — короткая**: описание 1–2 строки.
   - **Тест 05 — длинная**: описание 30+ строк (можно вставить произвольный длинный текст).

## 1. Короткое описание

- [ ] Открыть «Тест 05 — короткая».
- [ ] Поле «Описание» отображается высотой ~60–80 px.
- [ ] Внутреннего скролла **нет**.
- [ ] Текст полностью виден.

## 2. Длинное описание

- [ ] Открыть «Тест 05 — длинная».
- [ ] Поле «Описание» **растянулось целиком** под содержимое — все 30+ строк видны без внутреннего скролла.
- [ ] Видна полная прокрутка модалки/вкладки «Основное» (внешний скролл).
- [ ] Никакого «двойного» скролла (textarea внутри + вкладка снаружи) нет.

## 3. Печатание

- [ ] Открыть «Тест 05 — короткая». Кликнуть в поле «Описание».
- [ ] Начать вводить новые строки (нажимать Enter много раз).
- [ ] Поле растёт по мере добавления — НИКОГДА не появляется внутренний скролл.
- [ ] Удалить часть строк — поле уменьшается соответственно (но не ниже 60 px).

## 4. Переключение вкладок

- [ ] Открыть «Тест 05 — длинная».
- [ ] Перейти на «Чек-лист», затем вернуться на «Основное».
- [ ] Поле «Описание» сразу показывает корректную высоту, текст не обрезан.

## 5. Ручной resize

- [ ] В правом нижнем углу textarea — мини-«ушко» для ручного растяжения.
- [ ] Можно вручную растянуть выше расчётной высоты — работает.
- [ ] При следующем вводе/удалении символа высота подстраивается под содержимое.

## 6. Регрессия

- [ ] Превью описания на карточке доски (если есть) — обрезается одной строкой с «…» (как было).
- [ ] Сохранение задачи (кнопка «Сохранить») — текст описания записан целиком.
- [ ] Проверка истории — изменение описания корректно отображено.

## 7. Чтение чужой задачи (read-only)

- [ ] Открыть задачу, выданную мной подчинённому, как этот подчинённый (или иначе попасть в `readOnly`-режим).
- [ ] Поле «Описание» readonly, но всё равно тянется под содержимое.
