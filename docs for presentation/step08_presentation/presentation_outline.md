## Слайд 2 — Проблема

**Как сейчас ведётся учёт задач?**

- Устные договорённости — забываются
- Стикеры на мониторе — теряются
- Excel-таблицы — устаревают, нет общего доступа
- Внешние сервисы (Trello, Jira) — данные уходят за периметр предприятия, нет интеграции с PLM

**Результат:** задачи теряются, сроки срываются, руководитель не видит загрузку.

---

## Слайд 3 — Решение

**Канбан-доска внутри Союз-PLM**

- Визуальная доска с колонками: Надо сделать → В работе → Ожидание → Готово
- Карточки задач с приоритетами, сроками, исполнителями
- Перетаскивание мышкой (drag-and-drop)
- Работает прямо в стандартном клиенте PLM — ничего не нужно устанавливать

[СКРИНШОТ: Общий вид доски]

## Слайд 4 — Демонстрация: создание задачи

**Создание задачи за 5 секунд:**

1. Нажать «+» в любой колонке
2. Ввести название, выбрать приоритет и срок
3. Назначить исполнителя
4. Готово — карточка на доске

[СКРИНШОТ: Панель создания задачи]

---

## Слайд 5 — Демонстрация: работа с задачами

**Управление задачами:**

- Перетащить карточку в другую колонку = сменить статус
- Цветовая маркировка: красный (высокий приоритет), жёлтый (средний), зелёный (низкий)
- Просроченные задачи подсвечиваются красным
- Двойной клик — открыть полную карточку с описанием, вложениями и комментариями

[СКРИНШОТ: Карточка задачи]

## Слайд 6 — Уведомления

**Автоматические уведомления:**

- При назначении задачи — исполнитель получает сообщение в PLM
- При смене исполнителя — новый исполнитель получает сообщение в PLM
- Не нужно звонить и спрашивать — система сама оповестит

[СКРИНШОТ: Уведомление в PLM]

## Слайд 7 — Контроль для руководителей

**Начальник сектора / отделения видит:**

- Переключатель: «Мои задачи» / «Задачи сектора» / «Задачи отделения»
- Кто чем занят прямо сейчас
- Сколько задач просрочено
- У кого перегрузка, у кого свободные руки

## Слайд 8 — Отчёты

**Статистика по задачам:**

- Фильтр по периоду: неделя / месяц / квартал
- Сколько создано, выполнено, просрочено
- Таблица по исполнителям с прогресс-барами

## Слайд 9 — Преимущества для предприятия

| Критерий | Без канбана | С канбаном |
|----------|------------|-----------|
| Прозрачность | Никто не знает кто чем занят | Всё видно на доске |
| Контроль сроков | Узнаём о просрочке постфактум | Подсветка просроченных задач |
| Коммуникация | Звонки, письма, совещания | Автоматические уведомления |
| Безопасность | Данные в облачных сервисах | Всё внутри PLM |
| Отчётность | Ручной сбор данных | Автоматические отчёты |

## Слайд 10 — Техническая простота

**Не требует:**
- Отдельного сервера
- Установки ПО на рабочие места
- Лицензий на сторонние продукты
- Изменения существующей БД

**Установка:**
- Обновление `CURRENT_*` шаблонов и проверка обязательных атрибутов
- Работает в стандартном клиенте Союз-PLM
- Использует штатные механизмы БИС v3

## Слайд 12 — Итог

**Канбан-доска в Союз-PLM — это:**

1. Прозрачность работы для всех уровней
2. Контроль сроков и загрузки в реальном времени
