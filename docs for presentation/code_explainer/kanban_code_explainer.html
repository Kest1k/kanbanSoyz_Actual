runtime files
Какие файлы реально работают

Проект держится на шести runtime-файлах: один рисует экран, один рулит UX, один красит, два выполняют C#-логику, один открывает доску.

Главный диспетчер: `Invoke`, `BeforeRender`, CRUD, роли, отчёты, вложения, чат.

bridge
Главный мост JS ↔ C#

`window.external.InvokeTemplate()` — это местная “шина команд”, не AJAX и не REST.

kanban.jsСобирает строку параметров, валидирует UI, вызывает метод backend.
InvokeTemplate("MethodName", "a|b|c")
"MoveTask"

"CreateTask"
DoCreateTask()
"GetTaskDetails"
DoGetTaskDetails()
"GetReport"
DoGetReport()
"AddComment"
DoAddComment()
C# Invoke`switch(methodName)` выбирает `Do*`, работает с PLM и возвращает `OK`, JSON, ключ или `ERROR:*`.
Контракт хрупкий: если меняешь формат строки, меняй обе стороны сразу06

maintenance map
Куда идти, если нужно что-то поменять

Практическая навигация: начинай с симптома, потом открывай нужный слой.

Новая кнопка / поле
HTML id/class → window.function → CSS

Добавить DOM, экспортировать JS-функцию, при необходимости стилизовать.

Новое действие сервера
JS InvokeTemplate → C# case → Do*

Сразу зафиксировать формат payload и формат ответа.

Изменить карточку
BuildCardData → Liquid card → .kb-card

Данные карточки приходят из C#, внешний вид живёт в HTML/CSS.

Изменить модалку
GetTaskDetails ↔ tcmRender ↔ SaveTask

Поля должны совпадать в JSON, DOM id и save payload.

Изменить роли
GetHierarchyInfo + GetAllowedUserIdSet + _kbH

Фронт показывает варианты, backend принимает окончательное решение.

Только визуал
kanban.css: .kb-* / .tcm-* / .rpt-*

Если не меняешь DOM и payload, C# трогать не нужно.

Правило проекта: не ломать JS ↔ C# transport contract без синхронных изменений16
