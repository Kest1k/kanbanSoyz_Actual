KanbanScreen — Серверный скрипт экрана

Файл: SOYUZ_UPLOAD_KanbanScreen_script.cs Тип: C# серверный скрипт экрана PLM

Назначение

Главный серверный скрипт канбан-доски. Отвечает за:

  • Первичную загрузку задач и распределение по колонкам
  • Определение роли пользователя и уровня доступа
  • Обработку всех AJAX-запросов от клиента (создание, перемещение, удаление задач, отчёт, иерархия)

Структура скрипта

1. Инициализация и определение роли

// Определение роли текущего пользователя
var currentUser = Context.CurrentUser;
var roles = currentUser.Roles;

// Иерархия ролей (от высшей к низшей):
// КанбанНачОтдела → КанбанНачОтделения → КанбанНачСектора → обычный пользователь
var isHead = roles.Contains("КанбанНачОтдела");
var isDeptHead = roles.Contains("КанбанНачОтделения");
var isSectorHead = roles.Contains("КанбанНачСектора");
var isManager = isHead || isDeptHead || isSectorHead;

Роль определяет:

  • Видимость кнопки «Все задачи» и панели иерархии
  • Набор доступных фильтров (отделение/сектор/сотрудник)
  • Возможность назначать задачи другим пользователям

2. Загрузка задач (первичный рендер)

При первичной загрузке страницы скрипт:

  1. Получает все задачи шаблона KanbanTask через PLM API
  2. Фильтрует по текущему пользователю (режим «Мои задачи»)
  3. Сортирует: сверхсрочные → по дате создания (новые сверху)
  4. Распределяет по 4 коллекциям: col_0, col_1, col_2, col_3
  5. Для каждой задачи формирует объект с полями для шаблона

Поля объекта задачи для шаблона:

Поле Тип Описание
id string Идентификатор объекта задачи
title string Название задачи
details string Описание (обрезается до 100 символов)
priority string Приоритет: low, medium, high, urgent
dueDate string Срок в формате ДД.ММ.ГГГГ
completedDate string Дата завершения (только для колонки «Готово»)
assigneeName string ФИО исполнителя
initials string Инициалы для аватара (напр. «ИП»)
isOwner string "true" если текущий пользователь — создатель
attachmentCount int Количество вложений
commentCount int Количество комментариев

3. AJAX-обработчики

Все AJAX-запросы приходят с параметром ajaxAction. Скрипт возвращает JSON.

moveTask — Перемещение задачи

Параметры: taskId, newStatus
Действие: Меняет атрибут статуса задачи. Если newStatus=3 → ставит дату завершения.
Ответ: { "ok": true }

deleteTask — Удаление задачи

Параметры: taskId
Действие: Удаляет объект задачи из PLM (только если пользователь — создатель или руководитель).
Ответ: { "ok": true }

createTask — Создание задачи

Параметры: title, details, status, priority, dueDate, assigneeId, attachments (JSON)
Действие: Создаёт новый объект KanbanTask, заполняет атрибуты, привязывает вложения.
Ответ: { "ok": true, "id": "<новый_id>" }

createGroupTasks — Групповое создание

Параметры: title, details, status, priority, dueDate, userIds (JSON-массив), attachments (JSON)
Действие: Создаёт отдельную задачу для каждого пользователя из списка.
Ответ: { "ok": true, "count": N }

getHierarchy — Получение иерархии

Параметры: нет
Действие: Возвращает структуру отделений/секторов для текущего руководителя.
Ответ: { "departments": [...], "sectors": [...], "role": "..." }

getUsers — Получение списка пользователей

Параметры: deptId (опц.), sectorId (опц.)
Действие: Возвращает список пользователей с фильтрацией по подразделению.
Ответ: { "users": [{ "id": "...", "name": "...", "sectorName": "..." }, ...] }

loadBoard — Перезагрузка доски

Параметры: mode ("my"/"all"), deptId (опц.), sectorId (опц.), userId (опц.)
Действие: Возвращает задачи с учётом фильтров для динамической перезагрузки.
Ответ: { "col_0": [...], "col_1": [...], "col_2": [...], "col_3": [...] }

getReport — Данные отчёта

Параметры: period ("week"/"month"/"quarter"/"all"), scope (опц.)
Действие: Агрегирует статистику по задачам за период.
Ответ: { "summary": {...}, "rows": [...] }

searchObjects — Поиск объектов PLM

Параметры: query (строка поиска)
Действие: Ищет объекты PLM по имени для привязки как вложения.
Ответ: { "results": [{ "id": "...", "name": "...", "template": "..." }, ...] }

4. Вспомогательные методы

  • GetInitials(name) — извлекает инициалы из ФИО (напр. «Иванов Пётр» → «ИП»)
  • FormatDate(date) — форматирует дату в ДД.ММ.ГГГГ
  • IsOverdue(task) — проверяет просроченность задачи
  • GetTasksForUser(userId) — получает задачи конкретного пользователя
  • GetTasksForScope(deptId, sectorId) — получает задачи по подразделению
  • BuildTaskObject(task) — формирует объект задачи для JSON/шаблона

5. Переменные шаблона (Scriban)

Скрипт передаёт в HTML-шаблон следующие переменные:

Переменная Тип Описание
col_0 array Задачи колонки «Надо сделать»
col_1 array Задачи колонки «В работе»
col_2 array Задачи колонки «Ожидание»
col_3 array Задачи колонки «Готово»
current_user_id string ID текущего пользователя
current_user_name string Имя текущего пользователя
is_manager bool Является ли руководителем
user_role string Роль: head, deptHead, sectorHead, user

Безопасность

  • Удаление задачи разрешено только создателю или руководителю
  • Перемещение задачи разрешено только исполнителю или руководителю
  • Фильтры иерархии ограничены ролью пользователя (начальник сектора не видит чужие отделения)
  • Все входные параметры валидируются на сервере
