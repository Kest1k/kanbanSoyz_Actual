# Канбан-доска — Общий обзор проекта

## Назначение
Канбан-доска — веб-модуль системы **SOYUZ PLM**, реализующий управление задачами в формате Kanban.
Работает внутри PLM-платформы: серверная часть на **C# (Scriban-шаблоны)**, клиентская — **vanilla JS + CSS**, совместимые с **IE11**.

## Ключевые сущности

### Задача (KanbanTask)
- **Статус** (колонка): 0 = «Надо сделать», 1 = «В работе», 2 = «Ожидание», 3 = «Готово»
- **Приоритет**: `low`, `medium`, `high`, `urgent`
- **Поля**: название, описание, срок, исполнитель, создатель, дата создания, дата завершения, теги
- **Связи**: вложения (объекты/папки PLM), комментарии
- **SeenByList**: строка с ключами пользователей, просмотревших задачу (для бейджа «НОВАЯ»)

### Роли и иерархия
- **Обычный пользователь** — видит только свои задачи
- **Начальник сектора** — видит задачи своего сектора
- **Начальник отделения** — видит задачи отделения
- **Начальник отдела** — видит все задачи

Иерархия определяется через PLM-роли: `КанбанНачОтдела`, `КанбанНачОтделения`, `КанбанНачСектора`.

### Режимы просмотра
- **«Мои задачи»** — задачи текущего пользователя (по умолчанию)
- **«Все задачи»** — доступен руководителям, с фильтрами по отделению/сектору/сотруднику

## Основные функции

1. **Создание задач** — панель с полями: название, статус, приоритет, срок, описание, исполнитель
2. **Групповое создание** — одна задача нескольким исполнителям одновременно
3. **Drag-and-drop** — перетаскивание карточек между колонками (смена статуса)
4. **Карточка задачи** — модальное окно с редактированием всех полей
5. **Комментарии** — чат внутри карточки задачи
6. **Вложения** — привязка объектов и папок PLM к задаче
7. **Отчёт** — сводка по задачам с фильтрами по периоду и области
8. **Сверхсрочные задачи** — особый приоритет `urgent` с пульсирующей анимацией
9. **Просроченные задачи** — визуальная подсветка задач с истёкшим сроком
10. **Уведомления** — всплывающее RTF-уведомление при назначении задачи (шаблон `ExclamationKanban`) с кликабельной ссылкой на доску
11. **Бейдж «НОВАЯ»** — синий индикатор на карточках, назначенных другим пользователем и ещё не просмотренных; исчезает при открытии карточки
12. **Справка** — модальное окно с инструкцией для пользователей

## Технические ограничения
- **IE11**: нет CSS Grid, нет `var()`, нет `fetch()`, нет стрелочных функций, нет `let`/`const`
- Все стили с `-ms-` и `-webkit-` префиксами
- AJAX через `XMLHttpRequest`
- Шаблонизация через Scriban (Liquid-совместимый синтаксис)

## Взаимодействие компонентов

```
Пользователь → HTML (клик/drag) → kanban.js → AJAX → KanbanScreen/KanbanTask (C#) → PLM API → БД
                                                  ↓
                                            JSON-ответ → kanban.js → DOM-обновление
```

Серверные скрипты обрабатывают AJAX-запросы через параметр `ajaxAction`:
- `KanbanScreen`: `moveTask`, `deleteTask`, `createTask`, `getReport`, `getHierarchy`, `getUsers`, `getTaskCard`, `searchObjects`
- `KanbanTask`: `getCard`, `saveCard`, `deleteCard`, `getComments`, `addComment`, `deleteComment`, `getAttachments`, `addAttachment`, `removeAttachment`, `getRevisions`
