# Канбан-доска — Общий обзор проекта

## Назначение
Канбан-доска — встроенный модуль **SOYUZ PLM / BIS v3** для постановки, исполнения и контроля задач в формате Kanban.
Работает внутри клиента Союза: серверная часть реализована на **C# scripting**, клиентская — на **vanilla JS + CSS**, совместимых с **IE11**.

## Ключевые сущности

### Задача (`KanbanTask`)
- **Статус**: `Todo`, `InProgress`, `Waiting`, `Done`
- **Приоритет**: `low`, `medium`, `high`, `urgent`
- **Поля**: название, описание, срок, исполнитель, создатель, дата завершения, теги
- **Связи**: объекты PLM, контейнеры PLM, комментарии, история изменений
- **Доп. признаки**: просрочка, бейдж «НОВАЯ», ownership для удаления/полного редактирования

### Роли и иерархия
- **`regular`** — видит только свои задачи
- **`leadEngineer`** — роль ведущего инженера; определяется по `Commission.NameKey`, начинающемуся с `ved`, и работает в рамках своего сектора
- **`headOfSector`** — руководитель сектора, секторный scope
- **`headOfDept`** — руководитель отделения, scope отделения
- **`admin`** — полный организационный scope

Роль вычисляется не по общим PLM-ролям, а по связке пользовательских атрибутов:
- `Commission` — определяет тип должности
- `Context` — определяет организационный контекст (`510кт`, `600кт` и т.д.)

## Режимы просмотра
- **`my`** — только мои задачи
- **`all`** — все задачи, только для `admin`
- **`dept`** — всё отделение, для `headOfDept`
- **`sector`** — весь сектор, для `headOfSector` и `leadEngineer`
- **`user:KEY`** — задачи конкретного сотрудника
- **`group:CTX`** — задачи группы/подразделения по `Context`

Текущий режим хранится в `obj.PropertyBag["KbViewMode"]`.

## Основные функции

1. **Создание задач** — одиночная постановка себе или сотруднику в разрешённом scope.
2. **Групповое создание** — отдельная задача каждому выбранному исполнителю.
3. **Drag & drop** — изменение статуса карточек между колонками.
4. **Карточка задачи** — редактирование, история, чат, вложения.
5. **Комментарии** — обсуждение внутри задачи.
6. **Вложения** — привязка объектов и контейнеров PLM.
7. **Отчёт** — статистика по периоду и текущему scope.
8. **Серверная проверка назначения** — backend не позволяет назначить задачу вне допустимого подразделения даже при обходе UI.
9. **Кнопка открытия доски** — отдельный скрипт команды для перехода на экран канбана.

## Технические ограничения
- **IE11 / WebBrowser**: без `fetch`, `Promise`, `let`, `const`, стрелочных функций и современных DOM API
- Bridge клиент ↔ сервер: `window.external.InvokeTemplate()`
- HTML-шаблон рендерится на сервере через Liquid / DotLiquid
- В C#-скриптах Союза используются только реальные API платформы; вымышленные методы недопустимы

## Взаимодействие компонентов

```text
Пользователь
  → HTML / onclick / drag events
  → kanban.js
  → window.external.InvokeTemplate(methodName, params)
  → SOYUZ_UPLOAD_KanbanScreen_script.cs
  → PLM API / контейнеры / шаблоны / пользователи
  → строка / JSON
  → kanban.js
  → обновление DOM или RefreshBoard
```
