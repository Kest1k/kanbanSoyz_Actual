# Kanban-доска — Бэклог доработок (по обратной связи пользователей)

Дата актуальности: 29 апреля 2026

## 1. Критические исправления (Фаза 1 — делать в первую очередь)

### 1.1 Исправить работу комментариев в карточке задачи
- Кавычки в комментариях отображаются как `&quot;`
- Переносы строк (`\n`) не работают, текст «слипается»
- Добавить возможность **редактирования** своих комментариев
- Добавить **уведомления** (Exclamation) при появлении нового комментария (автору задачи и/или текущему исполнителю)

### 1.2 Добавить удобные горячие клавиши
- `F5` — обновить доску полностью
- `Esc` — закрыть любую открытую модалку, панель создания или карточку
- `Enter` — сохранить изменения в открытой карточке задачи
- `Ctrl + Enter` — отправить комментарий в поле чата

### 1.3 Убрать кнопку «+» из заголовков колонок
- Оставить кнопку «+» **только** в колонке «Надо сделать»
- Во всех остальных колонках («В работе», «Ожидание», «Готово») кнопку создать задачу убрать

### 1.4 Улучшить сортировку задач в колонках
- Новые задачи должны появляться **выше** старых
- Приоритет всегда имеет высший приоритет: `urgent → high → medium → low`
- Внутри одного приоритета — сортировка по дате создания (новые сверху)

## 2. Важные улучшения (Фаза 2)

### 2.5 Глобальный поиск по задачам
- Добавить поле поиска в верхнюю панель
- Поиск должен работать по названию, описанию, тегам и тексту комментариев
- Фильтрация должна происходить мгновенно по всем видимым колонкам

### 2.6 Фильтр по периоду времени
- Возможность фильтровать задачи по дате создания и/или дате завершения
- Пример: «с 01.01.2026 по 31.01.2026»
- Должен работать как на основной доске, так и в отчётах

### 2.7 Чекбоксы / подзадачи внутри задачи
- В карточке задачи добавить возможность создавать список подзадач с чекбоксами
- Подзадачи должны сохраняться и отображаться в карточке

### 2.8 Блокнот / заметки внутри задачи
- Отдельное большое текстовое поле «Заметки» или «Личный блокнот»
- Сохранение истории изменений (кто и когда писал)

## 3. Права и видимость задач (очень важный блок)

### 3.9 Полные права руководителя на задачи подчинённых
Руководитель (нач. сектора, ведущий инженер, нач. отделения, админ и выше) должен иметь **полные права** на любую задачу, созданную в его scope:
- Редактировать название, описание, срок, приоритет, исполнителя
- Перемещать drag & drop
- Удалять задачу
- Менять статус
- По сути — работать с задачей так, **как будто он сам её создал**

Обычные сотрудники могут полностью управлять только своими задачами.

### 3.10 Личные (приватные) задачи
При создании задачи добавить галочку **«Личная задача»**.
Такие задачи:
- Видит **только сам создатель**
- Ни один руководитель (даже генеральный конструктор, нач. сектора и т.д.) их **не видит**
- Они **не отображаются** в общей доске других пользователей
- Не учитываются в отчётах и статистике

## 4. Функционал для руководителей и отчётности

### 4.11 Функция принятия результата задачи
- При переводе задачи в колонку «Готово» она получает статус «Ожидает принятия»
- Только руководитель, который поручил задачу, может нажать кнопку **«Принять результат»**
- До принятия задача остаётся в «Готово» с соответствующим визуальным бейджем

### 4.12 Обратная связь руководителю о выполнении
- При переводе задачи в «Готово» (и особенно после принятия) отправлять уведомление руководителю, который создал/поручил задачу

### 4.13 Расширенный отчёт выполненных задач
- В отчёте добавить возможность просмотра выполненных задач за период
- Добавить колонку **«Принял»** (кто и когда принял результат)

## 5. Долгосрочные фичи (Фаза 4+)

### 5.14 Повторяемые (recurring) задачи
- Возможность создавать задачи, которые автоматически повторяются по расписанию

### 5.15 Автоматические гиперссылки из Директум
- При создании задачи через интеграцию с Директум автоматически подтягивать и сохранять ссылку на документ/поручение
