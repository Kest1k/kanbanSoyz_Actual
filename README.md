# Канбан-доска для Союз-PLM

Инструмент управления задачами, встроенный в BIS Союз-PLM как дополнительный экран. Работает без установки стороннего ПО — только файлы скрипта и HTML/CSS/JS, которые загружаются через стандартный интерфейс Союза.

## Возможности

- **Drag & drop** задач между колонками: Надо сделать / В работе / Ожидание / Готово
- **Карточка задачи** — описание, срок, приоритет, назначение исполнителя, история изменений
- **Ролевая иерархия** — сотрудник видит свои задачи, начальник сектора — сектор, начальник отделения — отдел, администратор — все
- **Отчёты** — выполнено / просрочено / в работе по исполнителю за период
- **Вложения** — прикрепление объектов и контейнеров PLM к задаче
- **Чат** — комментарии к задаче внутри карточки
- **Групповая постановка** — создать задачу сразу нескольким сотрудникам
- **IE11-совместимость** — работает в WebBrowser ActiveX control (BIS v3)

## Стек

| Слой | Технология |
|------|-----------|
| Backend | C# scripting в Союз-PLM BIS v3 |
| Frontend | HTML + CSS + JavaScript (ES5, IE11) |
| Bridge | `window.external.InvokeTemplate()` |
| Шаблонизатор | Liquid (серверный рендер HTML) |

## Структура

```
scripts/                 ← файлы для загрузки в Союз
  SOYUZ_UPLOAD_KanbanScreen_script.cs   серверный C# скрипт экрана
  SOYUZ_UPLOAD_KanbanTask_script.cs     скрипт шаблона задачи
  KanbanBoard_HTML.html                 HTML-шаблон экрана
  kanban.css                            стили (→ CSS/ на сервере)
  kanban.js                             логика (→ JS/ на сервере)

docs/                    ← техническая документация
  00–05_*.md             модульная документация по компонентам
  HTML/                  HTML-версия документации
```

## Развёртывание

1. Создать справочники `Ref_KanbanStatus`, `Ref_KanbanPriority` и шаблон `KanbanTask` с атрибутами
2. Скопировать `kanban.css` → `CSS/`, `kanban.js` → `JS/` на сервер
3. Загрузить `SOYUZ_UPLOAD_KanbanScreen_script.cs` в Script экрана
4. Загрузить `KanbanBoard_HTML.html` в HTML/Viewer экрана
5. Проверить `<base href>` в шаблоне — должен совпадать с адресом сервера

Или использовать файл `Kanban Конфигурация-1.0.0.2.pmszcfg` для импорта конфигурации.
