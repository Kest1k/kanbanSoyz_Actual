# Требования к конфигурации (pmszcfg)

Перед развёртыванием фичи «Подзадачи / чекбоксы» необходимо вручную обновить конфигурацию `Kanban Конфигурация-1.0.0.X.pmszcfg` через клиент Soyuz-PLM (Configuration GUI).

## InfoType `KanbanTask` — добавить атрибуты

| Атрибут | Тип | MaxLength | Назначение |
|---------|-----|-----------|------------|
| `SubtasksJSON` | `Text` | 1 МБ | JSON-массив подзадач |
| `SubtasksTotal` | `IntegerNumber` | — | Денормализованный счётчик: всего |
| `SubtasksDone` | `IntegerNumber` | — | Денормализованный счётчик: выполнено |

## Почему это вручную

Файл `.pmszcfg` — бинарный сериализованный пакет конфигурации (формат CPLMScripting). Изменяется только через нативный клиент Soyuz-PLM.

## Совместимость

C#-код устойчив к отсутствию атрибутов:
- `task.GetValue<int>("SubtasksTotal/Done")` обёрнут в `try/catch` → вернёт `0`.
- Чтение `SubtasksJSON` через `task.GetString(...) ?? ""` → пустая строка, парсер вернёт пустой список.

После добавления атрибутов первая мутирующая операция (`AddSubtask` / `ToggleSubtask` / `DeleteSubtask`) проинициализирует все три значения.

## Версия пакета

Поднять номер версии (например, `1.0.0.4`) и сохранить новый файл рядом с предыдущим.
