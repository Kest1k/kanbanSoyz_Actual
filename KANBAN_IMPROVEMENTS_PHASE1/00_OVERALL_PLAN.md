## 0. Контекст проекта (минимально необходимый)

Канбан-модуль реализован на стеке Soyuz-PLM (BIS v3) и состоит из 6 файлов в `scripts/`:

| Файл | Роль | Размер |
|------|------|--------|
| `SOYUZ_UPLOAD_KanbanScreen_script.cs` | Серверная логика экрана (C#, Invoke-диспетчер) | ~119 KB / ~2790 строк |
| `SOYUZ_UPLOAD_KanbanTask_script.cs` | OnBeforeSave + уведомления (Exclamation) | ~10 KB |
| `KanbanBoard_HTML.html` | UI-шаблон (DotLiquid) | ~44 KB |
| `kanban.css` | Стили | ~55 KB |
| `kanban.js` | Клиентская логика (IE11, ES5, IIFE) | ~95 KB |
| `kanbanButton.cs` | Кнопка-команда открытия экрана | ~2 KB |
| `ExclamationKanban.cs` | Скрипт уведомлений (Exclamation tray) | ~18 KB |

Жёсткие рамки, которые держим во всех планах ниже:

- **JS — только IE11-совместимый ES5.** Никаких `let/const/=>/fetch/Promise/class/template literals/destructuring/for…of`.
- **C#** — без `using`, через `Service.HandleException`, всегда `GetEditable()` перед записью атрибутов.
- **DotLiquid** — нельзя сравнивать строки по неравенству `!= "1"`, использовать `!= ""`.
- **Транспорт JS↔C#** — только `window.external.InvokeTemplate("Method", "p1|p2")`. Любое изменение контракта правится синхронно с обеих сторон.
- **Деплой** — в каждой задаче: один маленький шаг → один смоук-тест → один коммит. Никакого батчинга.
- **Коммуникация** — RU; код-комментарии можно RU/EN, держим стиль файла.

1. **`01_fix_comments`** — главный пользовательский раздражитель. `&quot;` и потеря переносов видны на каждом сеансе. Редактирование и уведомления о новом комментарии — давнее ожидание.

### 4.2. Ветка

`feature/kanban-phase1-quickfixes`. PR в master после прохождения всех smoke-тестов и обновления документации.

### 4.3. Глобальные риски Phase 1

- **DotLiquid** не поддерживает фильтр `where` по двум полям, поэтому новый сервер-сайдовый сорт делаем **только в C#**, в Liquid не лезем.
- **IE11** — все JS-доработки на ES5: `var`, обычные функции, `keyCode/which`, `XMLHttpRequest`-style вызовы через `InvokeTemplate` (он синхронен).
- **`HtmlEnc` на сервере + `tcmChatEsc` на клиенте** — двойной escape. Исправление должно уметь читать **исторические** комментарии (см. план 01).
- **Exclamation-канал**: переиспользуем существующий `OnBeforeSave` + `ExclamationKanban.cs`. Никаких новых каналов уведомлений в Phase 1.
- **`InvokeTemplate("…", q|status|…)`** — строки разделены `|`. Перед отправкой пользовательского ввода всегда делаем `replace(/\|/g, " ")`.
- Тесты на тестовом сервере критичны: после каждого коммита прогонять смоук-сценарий из соответствующего `plan.md`.
