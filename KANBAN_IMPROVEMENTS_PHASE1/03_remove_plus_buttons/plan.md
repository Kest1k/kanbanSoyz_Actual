> **Затронутые файлы:** `scripts/KanbanBoard_HTML.html`, `scripts/kanban.css` (опционально), `docs/05_KANBAN_HTML_MARKUP.md`.

## 1. Техническое описание задачи

В каждом из 4 заголовков колонок есть ссылка-«+», которая открывает панель создания задачи и автоматически выставляет статус для этой колонки.

### Текущая разметка

В `KanbanBoard_HTML.html`:
- [строка 218](scripts/KanbanBoard_HTML.html#L218) — колонка 0 (`Надо сделать`)
- [строка 284](scripts/KanbanBoard_HTML.html#L284) — колонка 1 (`В работе`)
- [строка 350](scripts/KanbanBoard_HTML.html#L350) — колонка 2 (`Ожидание`)
- [строка 413](scripts/KanbanBoard_HTML.html#L413) — колонка 3 (`Готово`)

Все 4 ссылки одинаковые:

```html
<a href="#" onclick="showCreateTask(N); return false;"
    style="color:rgba(255,255,255,0.8);text-decoration:none;font-size:15px;float:left;margin-right:4px;">+</a>
```

### Что делаем (по backlog)

> **Оставить кнопку «+» только в колонке «Надо сделать» (col_0).**

Удаляем 3 анкора в колонках 1, 2, 3. В колонке 0 — оставляем без изменений.

### Зачем

- В колонках `В работе`, `Ожидание`, `Готово` создание новой задачи семантически неверно: задача начинается со статуса «Надо сделать» по дефолту.
- В колонке `Готово` «+» вообще выглядит абсурдно — будто пользователь сразу создаёт «готовую» задачу.
- В навбаре всегда доступен пункт «Новая задача» — он покрывает 100% сценариев.

## 4. CSS — что и где менять (`kanban.css`)

В текущем коде специфичных стилей `.kb-col-header a` или `.kb-col-header > a` нет — анкор стилизован inline. После удаления изменений в CSS не требуется.

**Проверить визуально**, что заголовки col_1, col_2, col_3 без «+» выровнены так же, как в col_0 (или симметрично — если в col_0 «+» сдвигает текст вправо, разные колонки будут выглядеть несимметрично).

Если визуально это критично — рассмотреть **компенсирующий отступ** в col_1..col_3:

```css
.kb-col-header.kb-col-header--no-add {
    padding-left: 16px;  /* визуальная компенсация ширины «+» */
}
```

И добавить класс на `<div class="kb-col-header kb-col-header--no-add">` в col_1..col_3. Опционально, по результату визуального ревью.

## 6. Риски и технические ограничения

1. **DotLiquid синтаксис** `{% assign cN = col_N | size %}{{ cN }}` остаётся.
2. **IE11** — без изменений, чистый HTML.
3. **Регрессия UX**: пользователи, привыкшие к «+» в `В работе`, могут спросить — почему пропало. Митигация:
   - В Help (фича 02) явно указать, что создание задачи — через навбар «Новая задача» или из колонки «Надо сделать».
   - В презентации генеральному директору обосновать: «задача начинается с «Надо сделать», логика канбана».
4. **`pmszcfg`-снимок** — изменение HTML-шаблона попадает в конфигурационный файл. Перепаковать после деплоя.
5. **Функция `showCreateTask`** — НЕ удалять. Используется анкором в col_0 и пунктом навбара.
6. **`kb-cnt`** счётчик колонок выровнен через float/abs (см. CSS) — после удаления «+» не должен сдвинуться. Проверить.

## 8. Тестовые сценарии

### Сценарий 1 — визуальная регрессия

1. Открыть доску.
2. Сравнить: «+» виден только над `Надо сделать`. Над `В работе`, `Ожидание`, `Готово` — отсутствует.
3. Цвета и счётчики — без изменений.

### Сценарий 2 — функциональность

1. Клик по «+» в `Надо сделать` → открылась панель создания, статус по умолчанию `Надо сделать`.
2. Клик по «Новая задача» в навбаре → открылась панель создания.
3. В обоих случаях создание задачи проходит штатно.

### Сценарий 3 — выравнивание

1. На минимальной поддерживаемой ширине экрана: заголовки выровнены, нет «прыгания».
2. На обычной ширине: то же самое.

### Сценарий 4 — деплой

1. Залить новую версию `KanbanBoard_HTML.html` + snapshot.
2. После `Ctrl+F5` пользователи видят чистые заголовки.
